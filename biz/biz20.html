Finance Team AI OCR Project — English Discussion Case Study

This study session is based on a realistic company situation. A finance manager asks the team to experiment with an AI OCR system that can read numbers from images and perform simple calculations.

Study Goal

Understand how AI OCR works, discuss automation in finance, and practice English communication through realistic workplace dialogue.

1 The Manager's Request

Scene 1

Finance Manager: Our finance team spends a significant amount of time manually entering numbers from receipts and invoices. Every day we receive many documents, and someone has to type all the numbers into the accounting system. I recently heard that AI technology can read text directly from images, so I wonder if we could experiment with a simple OCR system.

Employee A: That sounds like a very interesting idea. Most of us do not have deep experience with AI development, but we might be able to create a simple prototype using free tools. If we start with a basic system that reads numbers and performs calculations, it could help us understand the potential.

Finance Manager: Exactly. I am not expecting a perfect product right now. What I really want is a small experiment that shows whether this idea is feasible for our team.

Korean Translation

재무팀장: 우리 재무팀은 영수증과 인보이스의 숫자를 직접 입력하는 데 많은 시간을 쓰고 있어요. 매일 수많은 문서를 처리해야 하고, 누군가는 그 숫자를 시스템에 입력해야 합니다. AI가 사진에서 글자를 읽을 수 있다는 이야기를 들었는데, 간단한 OCR 시스템을 만들어볼 수 있을까요?

직원 A: 흥미로운 아이디어입니다. 우리 대부분이 AI 전문가라고 할 수는 없지만, 무료 도구를 사용하면 간단한 프로토타입 정도는 만들 수 있을 것 같습니다. 숫자를 읽고 계산까지 하는 기본 시스템부터 시작해 볼 수 있을 것 같습니다.

재무팀장: 맞아요. 완벽한 제품을 기대하는 것은 아닙니다. 이 아이디어가 실제로 가능한지 확인하는 작은 실험이 필요합니다.

2 Team Discussion

Scene 2

Employee B: So the manager wants an OCR system that reads numbers from images. Before we start building anything, we should probably understand how OCR technology actually works. If we understand the process clearly, we can design a better solution.

Employee C: OCR stands for Optical Character Recognition. It is a technology that allows computers to detect and read text from images or scanned documents. Many companies already use OCR to digitize paper records.

Employee A: In our case, we only need a very simple version. The system should read a mathematical expression from an image and then calculate the result automatically.

Korean Translation

직원 B: 팀장님은 사진에서 숫자를 읽는 OCR 시스템을 원하십니다. 우리가 개발을 시작하기 전에 OCR 기술이 어떻게 작동하는지 이해하는 것이 좋겠습니다.

직원 C: OCR은 Optical Character Recognition의 약자입니다. 컴퓨터가 이미지나 스캔 문서에서 글자를 인식할 수 있게 하는 기술입니다.

직원 A: 우리에게 필요한 것은 아주 간단한 버전입니다. 사진 속 수식을 읽고 결과를 계산하는 시스템이면 충분합니다.

3 Choosing Free Tools

Scene 3

Employee B: Since this is just a prototype, we should probably use free and open-source tools. The company may not want to invest money until we know whether the idea works.

Employee C: I did some research and found a tool called PaddleOCR. It is a free AI system that reads text from images and works quite well even on normal computers.

Employee A: For the website, we can build a simple interface using a Python framework called Flask. It is lightweight and perfect for small prototypes like this.

Korean Translation

직원 B: 이 프로젝트는 실험 단계이기 때문에 무료 오픈소스 도구를 사용하는 것이 좋겠습니다.

직원 C: 조사를 해보니 PaddleOCR이라는 도구가 있습니다. 무료이고 이미지에서 글자를 인식하는 AI 시스템입니다.

직원 A: 웹페이지는 Flask라는 Python 프레임워크로 만들 수 있습니다. 간단한 웹 서비스를 만들기에 적합합니다.

4 The First Problem

Scene 4

Employee C: I tried running the program, but the system keeps failing when it tries to read the image file. The program says that the file cannot be opened, even though the image is clearly inside the folder.

Employee B: That is strange. Could you check the file name of the image? Sometimes file names or folder paths can cause unexpected errors.

Employee C: I think I found the problem. The file name of the image is written in Korean characters, and it seems the program cannot read it correctly.

Employee A: Yes, that happens sometimes with certain libraries like OpenCV. To avoid this issue, we should always use English file names for images and project files.

Korean Translation

직원 C: 프로그램을 실행했는데 이미지 파일을 읽을 수 없다는 오류가 계속 발생합니다.

직원 B: 파일 이름을 확인해 보셨나요? 파일 이름이나 경로 때문에 오류가 생길 수도 있습니다.

직원 C: 파일 이름이 한국어로 되어 있습니다. 그것 때문에 문제가 생긴 것 같습니다.

직원 A: 맞습니다. OpenCV 같은 라이브러리는 한글 파일 이름을 제대로 처리하지 못할 때가 있습니다. 그래서 영어 파일 이름을 사용하는 것이 좋습니다.

5 Reporting to the Manager

Scene 5

Finance Manager: So how is the experiment going? Were you able to build the OCR prototype that we discussed last week?

Employee A: Yes, we successfully created a simple prototype. The system can read a math expression from an image and calculate the result automatically.

Employee B: We also discovered a few technical challenges during development. For example, the program fails when image file names are written in Korean, so we switched to English file names.

Finance Manager: That is great progress. Even a simple prototype can help us understand how AI could improve our workflow in the future.

Korean Translation

재무팀장: 프로젝트는 어떻게 진행되고 있나요? 지난주에 이야기한 OCR 프로토타입을 만들었나요?

직원 A: 네, 간단한 프로토타입을 만들었습니다. 사진 속 수식을 읽고 계산 결과를 보여주는 시스템입니다.

직원 B: 개발 과정에서 몇 가지 기술적인 문제도 발견했습니다. 예를 들어 이미지 파일 이름이 한글이면 오류가 발생했습니다.

재무팀장: 좋은 진전입니다. 작은 프로토타입이라도 미래에 업무 자동화 가능성을 보여줄 수 있습니다.

Discussion Questions

  • Why does the finance team want an OCR system?
  • What advantages do free AI tools provide?
  • Why did the program fail when the file name was written in Korean?
  • How could AI automation change office work in the future?
  • Should companies build AI systems internally or buy commercial software?
