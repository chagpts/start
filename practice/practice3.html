3. 승인절차와 지급통제 실습

복지비 횡령과 근로소득세 횡령사례를 바탕으로, 우리 AI OCR 회계 프로그램에서 신청-승인-지급-사후검증 절차를 어떻게 설계해야 하는지 실습합니다.

사고 포인트

  • 엑셀 조작을 통해 과다 신청 수행
  • 승인절차가 있었지만 실질적인 검증이 부족함
  • 전산 입력 전 금액을 쪼개어 횡령함
  • 지출명령과 실제 송금 금액이 일치하지 않음

AI OCR 회계 프로그램 실습

  • OCR 추출 금액과 지급 승인 금액 자동 비교
  • 금액 변경 이력과 변경자 추적 기능 설계
  • 엑셀 업로드 자료와 시스템 원장 간 대사 규칙 만들기
  • 분할 입력 또는 비정상 송금 패턴 경고 규칙 작성하기

감사체크리스트 연계

  • 승인 전 원본 증빙 확인 절차 존재 여부
  • 지급금액과 승인금액 자동 일치 점검 여부
  • 수기 자료 업로드 후 검증 절차 존재 여부
  • 이상금액 분할처리 탐지 기능 존재 여부

실습 결론

승인절차는 형식만 있으면 의미가 없습니다. 원본 문서, 승인 정보, 지급 결과가 자동으로 서로 연결되어야 하며, 금액 변경과 분할 입력을 실시간으로 감지해야 합니다.
