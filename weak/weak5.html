5. 데이터베이스 보안 점검

데이터베이스는 회계 프로그램의 핵심 자산입니다. 거래내역, 전표, 계정과목, 결산정보가 저장되므로 데이터베이스 보안이 약하면 재무정보 유출과 무단 변경이 직접적으로 발생할 수 있습니다.

주요 취약점

  • DB 관리자 계정 공유 사용
  • 업무 계정에 과도한 조회·수정 권한 부여
  • 민감정보 평문 저장
  • 테스트 DB에 운영 데이터 복제 후 방치
  • 불필요한 외부 접속 허용

점검 항목

  • DB 계정 분리와 최소권한 적용 여부
  • 중요 테이블 접근통제 설정 여부
  • 개인정보·계좌정보 암호화 여부
  • 운영/개발/테스트 DB 분리 여부
  • 외부 접속 IP 제한 및 접속 이력 관리 여부

개선 방안

DB 계정은 용도별로 분리하고, 응용프로그램이 필요한 권한만 사용하도록 설정해야 합니다. 또한 민감정보는 저장 시 암호화하고, 테스트 환경으로 운영 데이터를 복제할 경우 반드시 비식별화 또는 접근 제한을 적용해야 합니다.
