4. 데이터 입력값 검증

회계 프로그램은 많은 입력 화면과 조회 조건을 포함하고 있어, 입력값 검증이 미흡하면 SQL Injection, XSS, 비정상 데이터 저장과 같은 취약점이 발생할 수 있습니다. 이는 시스템 오류뿐 아니라 데이터 유출 및 위변조로도 이어질 수 있습니다.

주요 취약점

  • 입력값 길이, 형식, 범위 검증 미흡
  • 특수문자 필터링 미적용
  • 쿼리 직접 연결 방식 사용
  • 화면 출력 시 스크립트 문자 미치환
  • 파일 업로드 제한 미흡

점검 항목

  • 서버 측 입력 검증 수행 여부
  • Prepared Statement 등 안전한 쿼리 방식 사용 여부
  • 입력값 허용 목록 기반 검증 여부
  • 출력 시 HTML 이스케이프 처리 여부
  • 업로드 파일 확장자 및 크기 제한 여부

개선 방안

입력값 검증은 화면단뿐 아니라 서버단에서도 반드시 수행해야 합니다. 또한 SQL 구문은 매개변수 방식으로 처리하고, 조회 조건·메모·첨부파일 기능까지 포함해 전체 입력 경로를 점검해야 합니다.
