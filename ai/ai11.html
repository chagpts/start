머신러닝 이해

머신러닝은 컴퓨터가 사람이 일일이 규칙을 입력하지 않아도, 데이터를 통해 패턴을 학습하고 예측이나 분류를 수행하도록 만드는 기술입니다. 쉽게 말해 과거 데이터를 보고 기준을 배우고, 새로운 데이터가 들어왔을 때 스스로 판단하도록 만드는 방식입니다.

1. 머신러닝이란 무엇인가

전통적인 프로그램은 사람이 규칙을 직접 작성합니다. 예를 들어 “금액이 100만 원 이상이면 승인 필요”처럼 조건을 코드로 작성합니다. 반면 머신러닝은 많은 데이터를 보고 어떤 특징이 결과와 관련 있는지 스스로 학습합니다.

머신러닝의 핵심은 정답을 외우는 것이 아니라, 새로운 데이터에도 적용 가능한 패턴을 배우는 것입니다.

2. 머신러닝이 필요한 이유

현실의 데이터는 복잡하고 예외가 많습니다. 스팸 메일 분류, 이상 거래 탐지, 문서 자동 분류, 수요 예측처럼 사람이 모든 규칙을 직접 작성하기 어려운 문제에서 머신러닝이 유용합니다.

업무 머신러닝 활용 예시
회계 이상 전표 탐지, 비용 계정 자동 분류
문서 처리 영수증, 계약서, 세금계산서 자동 분류
금융 사기 거래 탐지, 신용 위험 예측
마케팅 고객 이탈 예측, 추천 시스템
운영 수요 예측, 재고 최적화

3. 규칙 기반 방식과 머신러닝 방식

규칙 기반 방식은 사람이 기준을 직접 정하기 때문에 이해하기 쉽고 통제하기 좋습니다. 하지만 예외가 많아지면 규칙이 복잡해지고 유지보수가 어려워집니다. 머신러닝은 데이터에서 패턴을 학습하기 때문에 복잡한 문제에 강하지만, 데이터 품질에 크게 영향을 받습니다.

구분 규칙 기반 머신러닝
판단 기준 사람이 직접 작성 데이터를 통해 학습
장점 명확하고 설명이 쉬움 복잡한 패턴을 찾을 수 있음
단점 예외가 많으면 관리 어려움 데이터 품질이 낮으면 성능 저하
예시 100만 원 이상이면 승인 필요 과거 거래 패턴을 보고 이상 거래 예측

4. 머신러닝의 기본 흐름

머신러닝 프로젝트는 보통 문제 정의에서 시작합니다. 그 다음 데이터를 수집하고, 전처리하고, 모델을 학습시킨 뒤, 성능을 평가하고 실제 업무에 적용합니다.

문제 정의 → 데이터 수집 → 데이터 전처리 → 모델 학습 → 성능 평가 → 배포 → 운영 및 개선

단계 설명
문제 정의 무엇을 예측하거나 분류할지 명확히 정함
데이터 수집 문제와 관련된 실제 데이터를 모음
전처리 누락값, 오류값, 중복값, 형식 불일치를 정리
모델 학습 데이터를 바탕으로 패턴을 학습
성능 평가 새로운 데이터에서도 잘 작동하는지 확인
운영 실제 업무에 적용하고 성능을 지속 점검

5. 지도학습

지도학습은 입력 데이터와 정답 데이터가 함께 있는 상태에서 학습하는 방식입니다. 예를 들어 거래 데이터와 함께 “정상” 또는 “이상”이라는 정답이 붙어 있으면, 모델은 어떤 특징이 이상 거래와 관련 있는지 학습할 수 있습니다.

문제 유형 설명 예시
분류 데이터를 정해진 그룹으로 나눔 스팸/정상 메일, 정상/이상 거래
회귀 숫자 값을 예측함 매출 예측, 수요 예측, 가격 예측

6. 비지도학습

비지도학습은 정답이 없는 데이터에서 숨겨진 구조나 패턴을 찾는 방식입니다. 고객을 비슷한 행동 패턴끼리 묶거나, 문서들을 주제별로 나누는 데 사용할 수 있습니다.

방법 설명 예시
군집화 비슷한 데이터끼리 그룹화 고객 그룹 분류, 거래 유형 묶기
차원 축소 많은 변수를 핵심 정보 중심으로 줄임 데이터 시각화, 주요 특징 파악
이상 탐지 일반 패턴에서 벗어난 데이터를 찾음 비정상 거래, 이상 로그인 탐지

7. 강화학습

강화학습은 행동에 따른 보상과 벌점을 통해 더 나은 선택을 학습하는 방식입니다. 게임, 로봇 제어, 경로 최적화처럼 반복적인 의사결정이 필요한 문제에 사용됩니다. 일반적인 회계 데이터 분석에서는 지도학습과 비지도학습이 더 자주 사용됩니다.

8. 학습 데이터와 평가 데이터

머신러닝 모델이 정말 잘 작동하는지 확인하려면 데이터를 학습용과 평가용으로 나누어야 합니다. 학습 데이터는 모델이 패턴을 배우는 데 사용하고, 평가 데이터는 모델이 처음 보는 데이터에서 얼마나 잘 판단하는지 확인하는 데 사용합니다.

같은 데이터로 학습하고 평가하면 모델이 잘하는 것처럼 보일 수 있습니다. 하지만 실제로는 정답을 외운 것일 수 있으므로 반드시 평가 데이터를 따로 두어야 합니다.

9. 데이터 전처리

머신러닝에서 데이터 전처리는 매우 중요합니다. 잘못된 데이터, 누락된 값, 중복된 값, 서로 다른 형식의 데이터가 있으면 모델 성능이 떨어질 수 있습니다.

  • 누락값 처리
  • 중복 데이터 제거
  • 이상값 확인
  • 날짜, 금액, 코드 형식 통일
  • 문자 데이터를 숫자 형태로 변환
  • 학습에 필요한 변수 선택

10. 과적합

과적합은 모델이 학습 데이터에 너무 맞춰져서 새로운 데이터에서는 성능이 떨어지는 현상입니다. 시험 문제의 답만 외운 학생처럼, 학습 데이터에서는 높은 점수를 보이지만 실제 업무에서는 약할 수 있습니다.

구분 설명
좋은 학습 새로운 데이터에도 적용 가능한 패턴을 학습
과적합 학습 데이터의 세부 잡음까지 외움
해결 방법 평가 데이터 분리, 모델 복잡도 조절, 데이터 추가, 교차검증 사용

11. 성능 평가 지표

모델 성능을 평가할 때 정확도만 보면 위험할 수 있습니다. 예를 들어 전체 거래 중 99%가 정상 거래라면, 모델이 모든 거래를 정상이라고 예측해도 정확도는 99%가 됩니다. 하지만 이상 거래는 하나도 잡지 못합니다.

지표 의미 중요한 상황
정확도 전체 중 맞춘 비율 데이터가 균형 잡힌 경우
정밀도 이상이라고 예측한 것 중 실제 이상 비율 오탐을 줄이고 싶을 때
재현율 실제 이상 중 찾아낸 비율 미탐을 줄이고 싶을 때
F1-score 정밀도와 재현율의 균형 오탐과 미탐을 함께 고려할 때

12. 실무 적용 시 주의사항

  • 모델보다 먼저 문제 정의가 명확해야 한다
  • 데이터 품질이 낮으면 좋은 모델도 성능이 낮다
  • 정답 라벨 기준이 사람마다 다르면 학습이 흔들린다
  • 정확도 하나만으로 모델을 평가하면 안 된다
  • 운영 중 데이터 패턴이 바뀌면 성능이 떨어질 수 있다
  • 처음부터 완전 자동화보다 의사결정 보조로 시작하는 것이 안전하다

13. 머신러닝 적용 예시: 회계 이상 거래 탐지

회계 분야에서는 머신러닝을 사용해 비정상 전표나 이상 거래를 탐지할 수 있습니다. 과거 거래 데이터를 학습해 일반적인 패턴을 파악하고, 그 패턴에서 크게 벗어나는 거래를 검토 대상으로 표시합니다. 

예시 흐름

1. 과거 전표 데이터 수집
2. 거래일, 거래처, 금액, 계정과목, 승인자 정보 정리
3. 정상 거래와 이상 거래 라벨링
4. 모델 학습
5. 신규 전표에 대해 이상 가능성 점수 계산
6. 고위험 거래를 회계 담당자가 검토

14. 역할극으로 이해하는 머신러닝

머신러닝은 사람이 규칙을 만들지 않아도 알아서 판단하는 건가요?

완전히 알아서 하는 것은 아닙니다. 사람이 문제를 정의하고 데이터를 준비해야 합니다.

기획자:

데이터 분석가:
항상 그렇지는 않습니다. 이상 거래 탐지처럼 드문 사건은 정확도보다 재현율이 더 중요할 수 있습니다.

기획자:
실무에서 가장 중요한 것은 무엇인가요?

데이터 분석가:
모델 선택보다 문제 정의, 데이터 품질, 성능 검증이 더 중요합니다.

학습 정리

  • 머신러닝은 데이터를 통해 패턴을 학습하는 기술이다
  • 규칙 기반 방식은 사람이 기준을 만들고, 머신러닝은 데이터에서 기준을 배운다
  • 지도학습은 정답이 있는 데이터로 학습한다
  • 비지도학습은 정답 없이 데이터의 구조를 찾는다
  • 모델 성능은 학습 데이터가 아니라 새로운 데이터에서 평가해야 한다
  • 정확도만 보지 말고 정밀도, 재현율 등 업무 목적에 맞는 지표를 봐야 한다
