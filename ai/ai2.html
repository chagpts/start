OCR의 이해

OCR은 Optical Character Recognition의 약자로, 이미지나 PDF 안에 있는 글자를 컴퓨터가 읽을 수 있는 텍스트 데이터로 변환하는 기술입니다. 회계 프로그램에서는 영수증, 세금계산서, 거래명세서, 인보이스 같은 문서를 자동으로 읽어 날짜, 거래처, 금액, 부가세 등의 정보를 추출하는 데 사용됩니다.

• 회계 문서 OCR 적용 시 주의사항

1. OCR이란 무엇인가

OCR은 사람이 눈으로 읽는 문자를 컴퓨터가 인식하도록 만드는 기술입니다. 예를 들어 종이 영수증을 사진으로 찍으면 컴퓨터 입장에서는 단순한 이미지일 뿐입니다. OCR은 그 이미지 안에서 글자 영역을 찾고, 각 글자가 무엇인지 판단하여 텍스트로 변환합니다.

OCR의 핵심은 “이미지를 텍스트로 바꾸는 것”입니다. 하지만 실무에서는 단순 변환보다 정확하게 읽고, 필요한 데이터만 추출하는 것이 더 중요합니다.

2. OCR이 필요한 이유

회계 업무에서는 반복적으로 많은 문서를 확인하고 입력해야 합니다. 영수증의 날짜, 거래처명, 금액, 부가세를 사람이 하나씩 입력하면 시간이 오래 걸리고 입력 오류도 발생할 수 있습니다. OCR을 사용하면 반복 입력 업무를 줄이고, 회계 데이터를 더 빠르게 처리할 수 있습니다.

업무 OCR 적용 효과
영수증 입력 날짜, 가맹점명, 금액 자동 추출
세금계산서 확인 공급가액, 부가세, 사업자등록번호 자동 인식
인보이스 처리 거래처, 청구일, 결제금액 추출
문서 검색 이미지 PDF도 텍스트 검색 가능

3. 우리가 사용할 OCR 엔진: Tesseract

우리는 OCR 엔진으로 Tesseract를 사용할 예정입니다. Tesseract는 이미지에서 문자를 인식해 텍스트로 변환하는 오픈소스 OCR 엔진입니다. 한글과 영어 모두 사용할 수 있으며, 서버나 로컬 환경에 설치해 직접 운영할 수 있다는 장점이 있습니다.

항목 설명
엔진 이름 Tesseract OCR
사용 목적 이미지 또는 PDF 문서에서 텍스트 추출
장점 오픈소스, 로컬 실행 가능, 다양한 언어 지원
주의점 문서 품질과 전처리에 따라 인식률 차이가 큼

Tesseract를 잘 쓰려면 OCR 엔진 자체보다 입력 이미지를 얼마나 깨끗하게 준비하느냐가 중요합니다.

4. OCR 처리 흐름

OCR은 보통 아래 흐름으로 진행됩니다. 문서를 수집한 뒤 이미지 품질을 정리하고, Tesseract로 텍스트를 추출한 뒤, 필요한 회계 필드만 다시 구조화합니다.

문서 수집 → 이미지 전처리 → Tesseract OCR 실행 → 텍스트 정제 → 필드 추출 → 검증 → 회계 시스템 입력

5. 이미지 전처리가 중요한 이유

Tesseract는 선명하고 정렬된 문서에서 좋은 성능을 냅니다. 반대로 이미지가 흐릿하거나, 기울어져 있거나, 배경에 그림자가 있으면 숫자와 문자를 잘못 인식할 가능성이 높아집니다.

문제 발생 가능한 오류
기울어진 문서 줄 단위 인식 오류, 단어 분리 오류
흐릿한 이미지 0/O, 1/I, 5/S 등 문자 혼동
그림자 문자 일부 누락 또는 배경을 글자로 인식
낮은 해상도 작은 숫자, 소수점, 쉼표 인식 실패

6. 주요 전처리 방법

전처리는 OCR 인식률을 높이기 위해 이미지를 정리하는 과정입니다. 회계 문서에서는 특히 숫자와 날짜가 중요하기 때문에 텍스트가 선명하게 보이도록 만드는 것이 핵심입니다.

전처리 설명
Grayscale 색상을 제거하고 흑백 명암 정보 중심으로 변환
Thresholding 글자와 배경을 분리해 텍스트를 더 선명하게 만듦
Denoising 작은 점, 얼룩, 배경 노이즈 제거
Deskew 기울어진 문서를 수평으로 보정
Crop 문서 밖 배경이나 불필요한 영역 제거

7. Tesseract 기본 사용 예시

Tesseract는 명령어 방식으로도 사용할 수 있고, Python에서 pytesseract 라이브러리를 통해 사용할 수도 있습니다. 아래는 Python에서 이미지를 읽고 OCR을 실행하는 간단한 예시입니다. 

import pytesseract
from PIL import Image

image = Image.open("receipt.png")

text = pytesseract.image_to_string(
    image,
    lang="kor+eng"
)

print(text)

한글과 영어가 섞인 회계 문서라면 lang="kor+eng"처럼 언어를 함께 지정할 수 있습니다. 영수증에 영어 항목이나 카드사명이 함께 있는 경우 유용합니다.

8. Tesseract 설정에서 중요한 요소

Tesseract는 설정에 따라 결과가 달라질 수 있습니다. 특히 문서가 한 줄인지, 여러 줄인지, 표 형태인지에 따라 페이지 분할 방식인 PSM 설정을 조정해야 합니다.

설정 의미
lang 인식할 언어 설정. 예: kor, eng, kor+eng
psm 페이지 분할 방식. 문서 구조에 따라 변경
oem OCR 엔진 모드 설정
whitelist 인식할 문자 범위를 제한할 때 사용 
custom_config = "--oem 3 --psm 6"

text = pytesseract.image_to_string(
    image,
    lang="kor+eng",
    config=custom_config
)

9. OCR 결과의 한계

OCR은 완벽하지 않습니다. 특히 회계 문서에서는 숫자, 쉼표, 소수점, 통화기호가 중요하기 때문에 작은 인식 오류도 금액 오류로 이어질 수 있습니다.

OCR 결과 실제 의미 문제
1O,OOO 10,000 숫자 0과 영문 O 혼동
2026-O3-O1 2026-03-01 날짜 숫자 0 오인식
합 계 12 000 합계 12,000 공백과 쉼표 누락
VAT 1S00 VAT 1500 S와 5 혼동

10. OCR 이후 텍스트 정제

Tesseract가 추출한 텍스트는 바로 회계 데이터로 사용하지 않습니다. 먼저 불필요한 공백, 줄바꿈, 특수문자, 통화기호를 정리하고 자주 발생하는 오인식을 보정해야 합니다. 

raw_text = "T0TAL : 1O,OOO KRW"

clean_text = raw_text.replace("T0TAL", "TOTAL")
clean_text = clean_text.replace("O", "0")

print(clean_text)

OCR 후처리의 목표는 “읽힌 텍스트”를 “회계 시스템에 넣을 수 있는 데이터”로 바꾸는 것입니다.

11. 회계 문서에서 추출할 핵심 필드

회계 OCR에서 중요한 것은 문서 전체 텍스트가 아니라 업무에 필요한 필드를 정확히 뽑아내는 것입니다.

필드 설명
거래일 영수증 발행일, 세금계산서 작성일자
거래처 가맹점명, 공급자명, 판매자명
금액 총 결제금액 또는 청구금액
부가세 VAT, 세액, 부가가치세
사업자등록번호 공급자 또는 거래처 식별 정보
결제수단 카드, 현금, 계좌이체 등

12. OCR 결과 검증

회계 업무에서는 OCR 결과를 반드시 검증해야 합니다. 금액 합계가 맞는지, 날짜 형식이 올바른지, 사업자등록번호 형식이 맞는지, 같은 영수증이 중복 등록되지 않았는지 확인해야 합니다.

  • 총액 = 공급가액 + 부가세인지 확인
  • 날짜가 올바른 형식인지 확인
  • 금액에 문자나 잘못된 기호가 섞여 있지 않은지 확인
  • 사업자등록번호 자릿수가 맞는지 확인
  • 거래처명이 기존 거래처 DB와 매칭되는지 확인
  • 이미 등록된 문서와 중복되는지 확인
if total != supply_amount + vat:
    print("검증 필요: 금액 불일치")

13. Tesseract 사용 시 실무 팁

  • 한글 문서라면 한글 학습 데이터가 설치되어 있어야 한다
  • 한글과 영어가 섞인 문서는 kor+eng 설정을 사용한다
  • 영수증처럼 좁고 긴 문서는 PSM 설정을 테스트해야 한다
  • 표 형태 문서는 OCR만으로 구조가 잘 보존되지 않을 수 있다
  • 숫자만 필요한 영역은 whitelist로 숫자 중심 인식을 시도할 수 있다
  • OCR 결과는 반드시 정규표현식과 검증 로직으로 후처리한다

15. 역할극으로 이해하는 OCR

회계 담당자:
OCR을 쓰면 영수증을 자동으로 읽을 수 있나요?

개발자:
네. Tesseract를 사용하면 이미지 속 글자를 텍스트로 추출할 수 있습니다.

회계 담당자:
그럼 추출한 텍스트를 바로 회계 프로그램에 넣으면 되나요?

개발자:
바로 넣으면 위험합니다. OCR은 글자를 잘못 읽을 수 있기 때문에 정제와 검증이 필요합니다.

회계 담당자:
어떤 오류가 자주 생기나요?

개발자:
0을 O로 읽거나, 5를 S로 읽는 경우가 있습니다. 금액에서는 이런 오류가 특히 위험합니다.

회계 담당자:
정확도를 높이려면 어떻게 해야 하나요?

개발자:
이미지를 선명하게 만들고, 기울기를 보정하고, OCR 이후 검증 로직을 넣어야 합니다.

회계 담당자:
결국 OCR은 시작 단계군요.

개발자:
맞습니다. OCR은 문서를 읽는 시작점이고, 회계 데이터로 만드는 과정은 그 다음입니다.

학습 정리

  • OCR은 이미지 속 문자를 텍스트로 변환하는 기술이다
  • 우리는 OCR 엔진으로 Tesseract를 사용할 예정이다
  • Tesseract 성능은 이미지 품질과 전처리에 크게 영향을 받는다
  • OCR 결과는 반드시 텍스트 정제와 검증 과정을 거쳐야 한다
  • 회계 문서에서는 날짜, 거래처, 금액, 부가세의 정확성이 가장 중요하다

한 줄 핵심 정리
OCR은 문서를 읽는 기술이고, Tesseract는 그 읽기 작업을 수행하는 엔진입니다. 하지만 회계 자동화에서는 OCR 이후의 정제와 검증이 진짜 핵심입니다.
