머신 러닝 기초

머신 러닝은 컴퓨터가 규칙을 일일이 사람이 입력받는 대신, 데이터를 통해 패턴을 학습하고 예측하도록 만드는 기술입니다. 쉽게 말하면, “조건문을 사람이 다 작성하는 방식”에서 “데이터를 보고 스스로 기준을 만들어가는 방식”으로 바뀌는 것입니다.

1. 머신 러닝은 왜 필요한가

전통적인 프로그램은 사람이 규칙을 명확히 정의할 수 있을 때 강합니다. 하지만 현실의 데이터는 복잡하고 예외가 많습니다. 예를 들어 정상 거래와 이상 거래를 구분하거나, 메일을 일반 메일과 스팸 메일로 나누거나, 문서 이미지를 보고 종류를 분류하는 작업은 단순 규칙만으로 처리하기 어렵습니다. 이럴 때 머신 러닝은 데이터를 보고 규칙을 학습하는 방식으로 문제를 해결합니다.

머신 러닝의 핵심은 “정답을 외우는 것”이 아니라 “새로운 데이터에도 적용 가능한 패턴을 배우는 것”입니다.

2. 머신 러닝의 기본 흐름

머신 러닝 프로젝트는 보통 문제 정의, 데이터 수집, 전처리, 학습, 평가, 배포, 운영의 순서로 진행됩니다. 여기서 가장 중요한 것은 모델보다 데이터입니다. 데이터가 부정확하거나 편향되어 있으면 좋은 결과를 기대하기 어렵습니다.

단계 설명
문제 정의 무엇을 예측하거나 분류할지 명확히 정한다
데이터 수집 문제와 관련된 실제 데이터를 모은다
전처리 누락값, 오류값, 형식 불일치를 정리한다
모델 학습 데이터를 바탕으로 패턴을 학습한다
성능 평가 새로운 데이터에서도 잘 동작하는지 검증한다
배포 및 운영 실제 서비스에 연결하고 성능을 지속 점검한다

3. 지도 학습과 비지도 학습

머신 러닝은 학습 방식에 따라 여러 종류로 나눌 수 있습니다. 가장 많이 쓰이는 것은 지도 학습입니다. 지도 학습은 입력과 정답이 함께 있는 데이터를 사용합니다. 예를 들어 이메일 내용과 함께 “스팸” 또는 “일반”이라는 정답이 붙어 있으면 모델은 어떤 특징이 스팸과 관련되는지 학습할 수 있습니다.

비지도 학습은 정답이 없는 데이터에서 구조를 찾는 방식입니다. 고객을 비슷한 행동 패턴으로 묶거나, 문서들을 주제별로 군집화하는 데 사용될 수 있습니다. 강화 학습은 보상과 벌점을 통해 행동 전략을 학습하는 방식으로, 게임, 제어, 최적화 분야에서 자주 등장합니다.

학습 방식 특징 예시
지도 학습 정답이 있는 데이터를 학습 사기 거래 탐지, 수요 예측, 분류
비지도 학습 정답 없이 패턴과 구조를 탐색 고객 군집화, 문서 묶기
강화 학습 행동 결과의 보상을 통해 학습 게임 플레이, 경로 최적화

4. 학습 데이터와 평가 데이터

모델 성능을 올바르게 보려면 데이터를 학습용과 평가용으로 나누어야 합니다. 학습용 데이터는 모델이 패턴을 배우는 데 사용하고, 평가용 데이터는 모델이 처음 보는 데이터에서 얼마나 잘 작동하는지 확인하는 데 사용합니다. 만약 같은 데이터를 학습과 평가에 모두 쓰면, 모델이 정답을 외운 것처럼 보일 수 있어 성능을 과대평가하게 됩니다.

좋은 모델은 학습 데이터에서만 잘 맞는 모델이 아니라, 처음 보는 데이터에서도 안정적으로 성능을 내는 모델입니다.

5. 과적합이란 무엇인가

과적합은 모델이 학습 데이터의 세부 잡음까지 지나치게 외워서, 실제 새로운 데이터에서는 성능이 떨어지는 현상입니다. 예를 들어 시험 문제를 이해하지 않고 답만 외운 학생과 비슷합니다. 학습 점수는 높아 보이지만 응용 문제에서는 약합니다. 머신 러닝에서는 이 문제를 줄이기 위해 데이터를 충분히 확보하고, 검증 절차를 엄격히 하고, 너무 복잡한 모델을 무조건 쓰지 않는 것이 중요합니다.

6. 정확도만 보면 안 되는 이유

분류 문제에서는 정확도만으로 성능을 판단하면 위험할 수 있습니다. 예를 들어 전체 데이터의 95퍼센트가 정상이고 5퍼센트만 이상이라면, 아무 생각 없이 전부 정상이라고 예측해도 정확도는 95퍼센트가 됩니다. 하지만 이상 거래를 전혀 잡지 못하므로 실제로는 쓸모가 없습니다. 그래서 정밀도, 재현율, 균형 잡힌 성능 해석이 중요합니다.

지표 의미 중요한 상황
정확도 전체 중 맞춘 비율 데이터가 균형 잡힌 경우
정밀도 이상이라고 예측한 것 중 실제 이상 비율 오탐이 비용이 큰 경우
재현율 실제 이상 중 찾아낸 비율 놓침이 위험한 경우

7. 현업에서 머신 러닝을 쓸 때 주의할 점

  • 데이터 정의가 명확하지 않으면 모델 품질도 불안정해진다
  • 정답 라벨 기준이 사람마다 다르면 학습 방향이 흔들린다
  • 운영 중 데이터 분포가 바뀌면 성능이 떨어질 수 있다
  • 설명 가능성이 필요한 업무에서는 단순한 모델이 더 유리할 수 있다
  • 자동화보다 의사결정 보조 용도로 시작하는 것이 현실적일 때가 많다
