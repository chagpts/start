데이터 수집과 전처리

데이터 수집과 전처리는 AI, 머신러닝, OCR, 데이터 분석 프로젝트의 출발점입니다. 좋은 모델이나 좋은 분석 도구가 있어도 데이터가 부정확하거나 정리되어 있지 않으면 결과를 신뢰하기 어렵습니다. 따라서 데이터를 어떻게 모으고, 어떻게 정리하고, 어떻게 사용할 수 있는 형태로 만드는지가 매우 중요합니다.

이 문서에서 배우는 내용

• 데이터 수집의 개념과 목적

• 수집 가능한 데이터 종류

• 데이터 품질 점검 기준

• 전처리의 주요 작업

• 실무에서 자주 발생하는 오류와 해결 방법

1. 데이터 수집이란 무엇인가

데이터 수집은 분석이나 AI 학습에 필요한 원천 데이터를 모으는 과정입니다. 데이터는 시스템에서 자동으로 생성될 수도 있고, 문서, 엑셀, 이미지, 로그, 설문, 외부 API 등을 통해 수집될 수도 있습니다.

데이터 수집의 핵심은 “많이 모으는 것”이 아니라, 문제 해결에 필요한 데이터를 정확하고 일관되게 모으는 것입니다.

2. 데이터 수집이 중요한 이유

데이터 품질은 전체 프로젝트 결과에 직접적인 영향을 줍니다. 부정확한 데이터를 기반으로 모델을 만들면 모델도 부정확한 판단을 하게 됩니다. 분석 결과가 틀리면 잘못된 의사결정으로 이어질 수 있습니다.

문제 영향
누락된 데이터 분석 결과가 편향되거나 일부 상황을 반영하지 못함
중복 데이터 금액, 건수, 빈도가 실제보다 크게 계산될 수 있음
잘못된 형식 날짜, 금액, 코드가 제대로 처리되지 않음
오류 데이터 모델 학습 결과나 통계 분석 결과가 왜곡됨
출처 불명확 데이터 신뢰성과 감사 추적성이 낮아짐

3. 수집 가능한 데이터 종류

프로젝트 목적에 따라 수집해야 할 데이터는 달라집니다. 회계 자동화에서는 영수증, 세금계산서, 거래 데이터가 중요하고, 머신러닝 프로젝트에서는 정답 라벨이 포함된 학습 데이터가 중요할 수 있습니다.

데이터 종류 예시
정형 데이터 엑셀, CSV, 데이터베이스 테이블, ERP 거래 내역
반정형 데이터 JSON, XML, 로그 파일, 이메일 메타데이터
비정형 데이터 PDF, 이미지, 영수증 사진, 계약서, 음성, 자유 텍스트
시스템 로그 로그인 이력, 권한 변경 이력, 배치 작업 로그
외부 데이터 공공 데이터, API 데이터, 웹 데이터, 시장 데이터

4. 데이터 수집 방법

데이터는 다양한 방식으로 수집할 수 있습니다. 중요한 것은 수집 방법마다 장단점과 품질 문제가 다르다는 점입니다. 자동 수집은 효율적이지만 시스템 연계 오류를 점검해야 하고, 수동 입력은 유연하지만 입력 오류가 발생하기 쉽습니다.

수집 방법 설명 주의사항
시스템 추출 ERP, 회계 프로그램, DB에서 데이터를 내려받음 추출 기준일, 필터 조건, 컬럼 정의 확인
파일 업로드 엑셀, CSV, PDF, 이미지 파일을 업로드 파일 형식과 컬럼 구조 표준화 필요
API 연계 외부 시스템과 자동으로 데이터를 주고받음 인증, 오류 처리, 호출 제한 확인
OCR 수집 이미지나 PDF에서 텍스트를 추출 이미지 품질과 OCR 오류 검증 필요
수동 입력 사용자가 화면에서 직접 입력 입력값 검증과 필수값 체크 필요

5. 수집 전 정의해야 할 것

데이터를 모으기 전에 무엇을 위해 데이터를 사용할지 명확히 해야 합니다. 목적이 불명확하면 불필요한 데이터를 많이 모으거나, 정작 필요한 데이터가 빠질 수 있습니다.

  • 해결하려는 문제는 무엇인가
  • 필요한 데이터 항목은 무엇인가
  • 데이터 출처는 어디인가
  • 수집 주기는 어떻게 할 것인가
  • 데이터 기준일은 무엇인가
  • 개인정보나 민감정보가 포함되는가
  • 누가 데이터 품질을 책임지는가

6. 데이터 품질 기준

수집한 데이터는 바로 사용할 수 없습니다. 먼저 품질을 확인해야 합니다. 대표적인 품질 기준은 정확성, 완전성, 일관성, 유효성, 중복 여부, 최신성입니다.

품질 기준 의미 예시
정확성 값이 실제 사실과 일치하는가 거래금액이 실제 영수증 금액과 일치
완전성 필수값이 빠지지 않았는가 거래일, 거래처, 금액 누락 없음
일관성 같은 의미의 값이 동일한 방식으로 표현되는가 스타벅스, STARBUCKS, Starbucks 표기 통일
유효성 값이 허용 가능한 범위와 형식에 맞는가 날짜가 YYYY-MM-DD 형식인지 확인
중복성 같은 데이터가 반복 저장되지 않았는가 동일 영수증 중복 업로드 확인
최신성 데이터가 필요한 시점 기준으로 최신인가 퇴사자 목록이 최신 인사 정보와 일치

7. 데이터 전처리란 무엇인가

데이터 전처리는 수집한 원천 데이터를 분석이나 모델 학습에 사용할 수 있도록 정리하는 과정입니다. 원천 데이터에는 누락값, 중복값, 오탈자, 형식 불일치, 이상값 등이 포함될 수 있습니다. 이 문제를 해결하지 않으면 분석 결과와 AI 모델 성능이 떨어집니다.

데이터 전처리는 원천 데이터를 “그냥 모은 데이터”에서 “분석 가능한 데이터”로 바꾸는 과정입니다.

8. 전처리의 주요 작업

작업 설명 예시
결측값 처리 비어 있는 값을 제거하거나 대체 거래처명이 없으면 확인 필요 표시
중복 제거 같은 데이터가 여러 번 들어온 경우 정리 동일 날짜, 금액, 거래처 영수증 중복 제거
형식 통일 날짜, 금액, 코드 형식을 표준화 2026.03.01 → 2026-03-01
오류값 수정 잘못 입력되거나 잘못 인식된 값을 보정 1O,OOO → 10,000
이상값 확인 일반 범위를 벗어난 값을 검토 식비 5,000,000원 거래 확인
데이터 변환 분석에 맞게 값을 변환 문자형 금액을 숫자형으로 변환

9. 결측값 처리

결측값은 데이터가 비어 있는 상태를 의미합니다. 결측값을 무조건 삭제하면 데이터가 줄어들고, 무조건 대체하면 왜곡이 생길 수 있습니다. 따라서 결측값의 원인과 중요도를 먼저 확인해야 합니다.

처리 방법 설명 주의사항
삭제 결측값이 있는 행이나 컬럼을 제거 중요 데이터가 많이 사라질 수 있음
기본값 입력 알 수 없음, 0, 기타 등으로 표시 실제 의미와 맞는지 확인 필요
평균·중앙값 대체 숫자 데이터에서 대표값으로 대체 분포가 왜곡될 수 있음
수동 확인 중요 필드는 담당자가 직접 확인 시간은 걸리지만 정확도가 높음

10. 중복 데이터 처리

중복 데이터는 분석 결과를 왜곡할 수 있습니다. 예를 들어 같은 영수증이 두 번 업로드되면 비용이 실제보다 크게 계산됩니다. 중복 여부는 단순히 파일명이 같은지만 보지 않고, 날짜, 거래처, 금액, 문서번호 등을 함께 비교해야 합니다. 

중복 확인 예시

1. 동일 날짜 + 동일 거래처 + 동일 금액 확인
2. 동일 세금계산서 번호 확인
3. 동일 파일 해시값 확인
4. 동일 사용자가 짧은 시간 안에 같은 문서를 업로드했는지 확인

11. 형식 통일

데이터 형식이 제각각이면 분석과 자동화가 어려워집니다. 특히 날짜, 금액, 사업자등록번호, 전화번호, 계정코드처럼 규칙이 있는 데이터는 표준 형식으로 통일해야 합니다.

항목 원본 예시 표준 형식
날짜 2026.03.01, 26/03/01 2026-03-01
금액 ₩12,000, 12,000원 12000
사업자등록번호 1234567890 123-45-67890
거래처명 스타벅스, STARBUCKS 스타벅스

12. 이상값 처리

이상값은 일반적인 범위에서 크게 벗어난 값입니다. 이상값은 단순 오류일 수도 있고, 실제 중요한 사건일 수도 있습니다. 따라서 무조건 삭제하기보다 원인을 확인해야 합니다.

  • 입력 오류인지 확인
  • 단위가 잘못 적용되었는지 확인
  • 실제 예외 거래인지 확인
  • 분석 목적상 제외할지 유지할지 결정
  • 제외했다면 근거를 기록

13. 텍스트 데이터 전처리

문서, OCR 결과, 고객 의견, 이메일 같은 텍스트 데이터는 별도의 정리가 필요합니다. 불필요한 공백, 특수문자, 줄바꿈, 오탈자를 정리하고 분석 목적에 맞게 단어 또는 문장 단위로 변환해야 합니다.

작업 예시
공백 정리 합 계 → 합계
특수문자 제거 ₩12,000원 → 12000
오탈자 보정 T0TAL → TOTAL
표기 통일 VAT, 부가세, 세액 → 부가세

14. 데이터 전처리 예시

원본 데이터

날짜: 2026.03.01
거래처: STARBUCKS
금액: ₩12,000
부가세: 1,O91
파일명: receipt.jpg

전처리 후

date: 2026-03-01
vendor: 스타벅스
amount: 12000
vat: 1091
file_name: 2026-03-01_스타벅스_12000.jpg

15. 전처리 결과 검증

전처리가 끝난 뒤에는 결과가 정확한지 검증해야 합니다. 전처리 과정에서 데이터가 잘못 변환되거나 중요한 정보가 사라질 수 있기 때문입니다.

  • 변환 전후 데이터 건수가 일치하는지 확인
  • 필수 컬럼에 결측값이 남아 있는지 확인
  • 금액 합계가 전처리 전후로 크게 달라지지 않았는지 확인
  • 날짜 범위가 예상 범위 안에 있는지 확인
  • 중복 제거 기준이 과도하지 않았는지 확인
  • 삭제하거나 수정한 데이터의 근거를 기록

16. 역할극으로 이해하는 데이터 수집과 전처리

기획자:
데이터만 많이 모으면 AI 모델을 만들 수 있나요?

데이터 분석가:
많이 모으는 것보다 정확하고 필요한 데이터를 모으는 것이 더 중요합니다.

기획자:
수집한 데이터를 바로 모델에 넣으면 안 되나요?

데이터 분석가:
바로 넣으면 위험합니다. 누락값, 중복값, 오류값, 형식 불일치를 먼저 정리해야 합니다.

기획자:
전처리는 단순히 보기 좋게 정리하는 건가요?

데이터 분석가:
아닙니다. 분석과 학습이 가능한 형태로 데이터를 바꾸는 과정입니다.

기획자:
가장 조심해야 할 부분은 무엇인가요?

데이터 분석가:
전처리 과정에서 중요한 데이터를 잘못 삭제하거나 왜곡하지 않는 것입니다.

학습 정리

  • 데이터 수집은 문제 해결에 필요한 데이터를 정확히 모으는 과정이다
  • 전처리는 원천 데이터를 분석 가능한 형태로 정리하는 과정이다
  • 데이터 품질은 정확성, 완전성, 일관성, 유효성, 중복성, 최신성으로 점검한다
  • 결측값, 중복값, 오류값, 이상값은 반드시 검토해야 한다
  • 전처리 후에는 데이터 건수, 합계, 필수값, 형식이 정상인지 확인해야 한다

한 줄 핵심 정리
데이터 수집과 전처리는 AI와 분석의 기반이며, 좋은 결과는 좋은 데이터 준비에서 시작됩니다.
